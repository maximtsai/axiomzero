# Recent Changes - Neural Tree & Burn System

## 1. Upgrade Tree Additions (js/nodeDefs.js)
- **Laser Node Extension**: Built out the upgrade path for the Laser weapon.
    - Added **DURATION** (`laser_duration`): +1 second to laser duration (Max 5 levels).
    - Added **PHOTON SATURATION** (`laser_aperture`): Increases beam width/thickness (Max 1 level).
    - Added **INCENDIARY** (`laser_incendiary`): Sets enemies on fire for 5 seconds (Max 3 levels).
    - Added **TWIN BEAMS** (`laser_twin_beams`): Increases beam count to two (Max 1 level, stub).
- **Positioning**: Fixed node layout using grid units (1.5 units left for DURATION, 1 unit above for status/count nodes).

## 2. Burn Status System (js/enemies/enemy.js)
- **EnemyModel Lifecycle**: Added `burnDuration`, `burnDamage`, and `burnTimer` to the model.
- **Pooling Safety**: Ensured periodic damage state is completely reset during enemy activation and deactivation to prevent pool-recycling bugs.
- **Centralized API**: Implemented `applyBurn(duration, damage)` as the standard helper for weapons to inflict damage-over-time.
- **Proxies**: Exposed burn properties on the `Enemy` controller for backward compatibility with existing manager loops.

## 3. Scalable DoT Processing (js/enemyManager.js)
- **Burn Ticking**: Added a highly scalable update check to the main enemy loop.
- **Performance**: Used a single per-enemy accumulator (`burnTimer`) and real-world seconds (`dt`) to process damage exactly once per second, avoiding heavy timer callbacks.

## 4. Visual Optimization (js/util/floatingText.js)
- **Frame Throttling**: Implemented `MAX_PER_FRAME = 15` cap to prevent visual noise and CPU spikes when hundreds of enemies take status damage simultaneously.
- **Increased Capacity**: Raised global `MAX_ACTIVE` limit from 120 to 200.
- **Memory Warming**: Updated the ObjectPool to pre-allocate 50 text instances on startup and increased the recycle capacity to 200.
